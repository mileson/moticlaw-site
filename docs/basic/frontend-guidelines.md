# 前端开发规范

## 适用范围

适用于官网首页、下载弹窗、语言/主题切换和公开安装入口等所有用户可见前端界面。

## 界面结构

- 首页由 `src/components/moticlaw-landing.tsx` 统一承载，页面级逻辑不在多个文件里分散实现。
- 首屏版本信息由服务端预取最新发布清单后注入，客户端仅负责刷新最新状态，不先渲染本地 fallback。
- 下载弹窗按 `macos / windows / linux` 三组展示平台项，平台可见性由实际 OSS 清单决定。

## 交互规范

- 下载入口优先推荐当前设备匹配的安装包，并提供其它平台的公开下载项。
- Windows 官网下载只展示安装版 `.exe`，不向普通用户暴露 portable 包。
- 用户可见文案应面向普通用户，避免暴露实现细节；安装提示需按平台给出准确说明。
- 页面和下载弹窗使用同一份发布数据模型，避免同屏出现不同版本或不同平台判断结果。

## 维护规则

- 每次新增界面模式、组件规范或交互规则后，必须检查并更新本文件。
