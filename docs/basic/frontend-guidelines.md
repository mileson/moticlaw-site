# 前端开发规范

## 适用范围

适用于官网首页、下载弹窗、语言/主题切换和公开安装入口等所有用户可见前端界面。

## 界面结构

- 首页由 `src/components/moticlaw-landing.tsx` 统一承载，页面级逻辑不在多个文件里分散实现。
- 首屏版本信息由服务端预取最新发布清单后注入，客户端仅负责刷新最新状态，不先渲染本地 fallback。
- 下载弹窗当前只展示 `macos / windows` 两组公开平台项；已下架的平台不能以前端“不可用”占位的形式继续暴露。

## 交互规范

- 下载入口优先推荐当前设备匹配的安装包，并提供其它平台的公开下载项。
- Windows 官网下载默认展开 x64 和 ARM64 安装版 `.exe` 入口，不向普通用户暴露 portable 包。
- 官网公开下载入口只暴露当前仍在售卖/分发的安装渠道；如 Linux 渠道下架，必须同时在数据层和展示层移除相关入口。
- 首屏默认语言和客户端挂载后的语言回填必须复用 `src/lib/locale.ts` 的同一套判定逻辑，并按浏览器语言优先级顺序选取当前支持的第一种语言。
- 用户手动切换过语言后，应优先沿用本地已保存的语言选择，避免后续访问被浏览器语言覆盖。
- 用户可见文案应面向普通用户，避免暴露实现细节；安装提示需按平台给出准确说明。
- 页面和下载弹窗使用同一份发布数据模型，避免同屏出现不同版本或不同平台判断结果。

## 维护规则

- 每次新增界面模式、组件规范或交互规则后，必须检查并更新本文件。
