# 项目文件结构

## 项目定位

当前项目是 MotiClaw 的独立官网仓库，负责首页展示、下载弹窗、发布清单接口、公开安装脚本和 Vercel 自动部署入口。

## 核心目录

- `src/app/`：Next.js App Router 路由入口、根布局、首页页面与全局样式。
- `src/components/`：首页主组件与下载交互视图。
- `src/lib/`：语言检测和 OSS 发布清单转换工具。
- `public/`：图标、品牌素材、安装脚本与静态发布清单镜像。
- `docs/basic/`：项目基础事实文档，描述前端、后端、流程和技术栈。
- `.openprd/`、`.codex/`、`.claude/`、`.cursor/`：OpenPrd 与多 Agent 工具链的项目级控制文件。

## 文件组织规则

- 新增文件时，应同步确认所在文件夹说明书是否需要更新。
- 跨模块移动文件时，应更新本文件中的目录结构和职责说明。
- 官网发布数据只允许通过 `src/lib/release-manifest.ts` 统一转换后进入页面层，避免在多个文件里直接解析 OSS JSON。

## 维护规则

- 每次新增、删除、移动目录或核心文件后，必须检查并更新本文件。
- 本文档只记录项目结构事实，不承载具体功能需求细节。
