# 产品逻辑说明

## 问题与目标

官网需要用简洁清晰的方式介绍 MotiClaw，并为用户提供可信、最新、可直接下载的桌面安装入口。目标用户包括准备首次了解产品的人、需要下载安装桌面端的人，以及确认当前发布版本和支持平台的人。

## 用户故事

- 作为首次访问官网的用户，我希望快速理解 MotiClaw 是什么，从而判断它是否适合我。
- 作为准备安装桌面端的用户，我希望官网能自动推荐当前仍公开提供的安装包，从而直接进入可用的下载路径。
- 作为已经明确平台的用户，我希望下载弹窗里只出现当前官网仍公开提供的安装项，从而避免点击到已经下架的渠道。
- 作为关注版本更新的用户，我希望官网展示的是 OSS 上最新公开发布版本，从而避免下载过期或未公开的安装包。

## 功能范围

- 当前版本包含：首页产品介绍、语言切换、主题切换、下载弹窗、OSS 最新发布清单接口、公开安装脚本入口、GitHub 联系入口。
- 当前版本包含：根据浏览器平台推荐安装包、展示 macOS/Windows 平台下载项、Windows 仅展示安装版 `.exe`。
- 当前版本不包含：站内账号系统、支付、下载统计后台、自定义版本选择器、旧版本归档浏览。

## 验收标准

- 首页首屏渲染时应直接显示最新 OSS 公开版本，而不是先显示旧版本后再切换。
- `/api/releases/latest` 必须返回来自 `https://moticlaw.oss-cn-hangzhou.aliyuncs.com/desktop/releases/latest.json` 的统一发布模型。
- 下载弹窗中只展示 macOS、Windows 公开支持项，Windows 仅出现 x64/ARM64 两个安装版 `.exe`，不出现 portable。
- 官网对外返回的发布模型与下载弹窗都不得暴露已下架的 Linux 安装渠道。
- 公共安装脚本默认读取官方 OSS `latest.json`，不依赖旧 OSS 域名或 GitHub Release 资产。

## 维护规则

- 每次需求边界、用户故事、验收标准发生变化后，必须检查并更新本文件。
